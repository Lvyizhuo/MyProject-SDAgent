### 1. 简单介绍一下事件循环机制，特别是微任务和宏任务的执行顺序。

**参考答案**：

**事件循环（Event Loop）** 是 JavaScript 处理异步操作的核心机制。

**基本概念**：
- **调用栈（Call Stack）**：同步任务执行，后进先出
- **任务队列（Task Queue）**：存放异步任务回调
  - **宏任务（Macro Task）**：`setTimeout`/`setInterval`、`setImmediate`、I/O、`requestAnimationFrame`、UI 渲染
  - **微任务（Micro Task）**：`Promise.then/catch/finally`、`async/await`、`process.nextTick`（Node.js）、`queueMicrotask`

**执行顺序**：
```
1. 执行同步代码（调用栈清空）
2. 执行所有微任务队列（清空微任务）
3. 执行一个宏任务
4. 再次执行所有微任务
5. 重复步骤 3-4
```

**经典例子**：
```javascript
console.log('1');

setTimeout(() => console.log('2'), 0);

Promise.resolve().then(() => console.log('3'));

console.log('4');

// 输出：1, 4, 3, 2
```

**执行过程解释**：
1. 同步执行：输出 `1`、`4`
2. 微任务：执行 `Promise.then`，输出 `3`
3. 宏任务：执行 `setTimeout`，输出 `2`

### 2. 在 AI 应用中经常用到流式输出，请说明 SSE（Server-Sent Events）和 WebSocket 的区别。

**参考答案**：

**SSE（Server-Sent Events）** 和 **WebSocket** 都是用于实现服务器向客户端推送数据的技术，但有明显区别。

**对比表**：

| 特性 | SSE | WebSocket |
|------|-----|-----------|
| **通信方向** | 单向（服务器 → 客户端） | 双向（服务器 ↔ 客户端） |
| **协议** | HTTP（长连接） | 独立协议（ws:// 或 wss://） |
| **数据格式** | 纯文本（可传输 JSON） | 二进制或文本 |
| **自动重连** | 原生支持（`retry` 字段） | 需要手动实现 |
| **连接数限制** | 浏览器同域名限制 6 个 | 无严格限制 |
| **适用场景** | 流式输出、实时通知、行情推送 | 实时聊天、在线协作、游戏 |

**在本项目中的实现**（ChatWindow.jsx）：

**SSE 流式解析**：
```javascript
const consumeSseEvents = (chunk, onData) => {
    const events = chunk.split('\n\n');
    const rest = events.pop() || '';

    for (const event of events) {
        const lines = event.split('\n');
        const dataLines = lines
            .filter(line => line.startsWith('data:'))
            .map(line => line.replace(/^data:\s?/, ''));
        if (dataLines.length > 0) {
            onData(dataLines.join('\n'));
        }
    }
    return rest;
};
```

**前端读取流**：
```javascript
const reader = response.body.getReader();
const decoder = new TextDecoder();
while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });
    buffer = consumeSseEvents(buffer, (data) => {
        // 处理 delta 文本增量
    });
}
```

**什么时候选 SSE**：
- 只需要服务器推送（AI 流式输出、实时通知）
- 想要简单的 API，不需要复杂的双向通信
- 需要自动重连机制

**什么时候选 WebSocket**：
- 需要双向通信（实时聊天、在线白板）
- 需要频繁的客户端→服务器消息
- 需要传输二进制数据

### 3. 如何实现一个带有并发控制的 Promise 控制器？

**参考答案**：

**并发控制**：限制同时执行的 Promise 数量，避免资源耗尽。

**场景**：同时上传 100 个文件，但只允许 3 个并发。

**实现方案**：

```javascript
class ConcurrencyController {
    constructor(maxConcurrent = 3) {
        this.maxConcurrent = maxConcurrent;  // 最大并发数
        this.current = 0;                     // 当前执行数
        this.queue = [];                      // 等待队列
    }

    // 添加任务
    add(task) {
        return new Promise((resolve, reject) => {
            this.queue.push({ task, resolve, reject });
            this.run();  // 尝试执行
        });
    }

    // 执行任务
    run() {
        // 还有空位且有等待任务
        while (this.current < this.maxConcurrent && this.queue.length > 0) {
            const { task, resolve, reject } = this.queue.shift();
            this.current++;

            task()
                .then(resolve)
                .catch(reject)
                .finally(() => {
                    this.current--;
                    this.run();  // 执行下一个
                });
        }
    }
}

// 使用示例
const controller = new ConcurrencyController(3);

const tasks = Array.from({ length: 10 }, (_, i) => () =>
    new Promise(resolve => {
        console.log(`开始任务 ${i}`);
        setTimeout(() => {
            console.log(`完成任务 ${i}`);
            resolve(i);
        }, Math.random() * 1000);
    })
);

// 批量添加
Promise.all(tasks.map(task => controller.add(task)))
    .then(results => console.log('所有完成:', results));
```

**更简洁的方案**：
```javascript
async function concurrentPool(tasks, poolSize = 3) {
    const results = [];
    const executing = [];

    for (let i = 0; i < tasks.length; i++) {
        const promise = tasks[i]().then(result => {
            results[i] = result;
        });
        executing.push(promise);

        // 达到并发限制，等待任意一个完成
        if (executing.length >= poolSize) {
            await Promise.race(executing);
            // 清理已完成的
            executing.splice(executing.findIndex(p => p === promise), 1);
        }
    }

    await Promise.all(executing);
    return results;
}
```

**在 AI 应用中的应用场景**：
- 批量文档嵌入（限制同时调用 Embedding API 的数量）
- 批量图片上传
- 并发调用 Rerank API

### 4. 谈谈 TypeScript 中的泛型，以及 extends 关键字在泛型中的作用。

**参考答案**：

**泛型（Generics）**：创建可复用、类型安全的组件，支持多种类型而不重复代码。

**基本用法**：
```typescript
// 简单泛型函数
function identity<T>(arg: T): T {
    return arg;
}

identity<string>('hello');  // 显式指定
identity(123);              // 类型推断

// 泛型接口
interface Box<T> {
    value: T;
}
const numberBox: Box<number> = { value: 42 };

// 泛型类
class Stack<T> {
    private items: T[] = [];
    push(item: T) { this.items.push(item); }
    pop(): T | undefined { return this.items.pop(); }
}
```

**extends 关键字在泛型中的作用**：

**作用 1：泛型约束（限制类型范围）**
```typescript
// 限制 T 必须有 length 属性
interface Lengthwise {
    length: number;
}

function logLength<T extends Lengthwise>(arg: T): void {
    console.log(arg.length);
}

logLength('hello');    // ✓ 字符串有 length
logLength([1, 2, 3]); // ✓ 数组有 length
// logLength(123);     // ✗ 数字没有 length
```

**作用 2：条件类型（三元判断）**
```typescript
// 如果 T 是 string 类型，返回 number，否则返回 boolean
type StringToNumber<T> = T extends string ? number : boolean;

type A = StringToNumber<string>;  // number
type B = StringToNumber<number>;  // boolean
```

**作用 3：泛型默认类型**
```typescript
// T 默认是 string
interface ApiResponse<T = string> {
    data: T;
    code: number;
}

const res1: ApiResponse = { data: 'success', code: 200 };
const res2: ApiResponse<{ id: number }> = { data: { id: 1 }, code: 200 };
```

**在 AI 项目中的实际应用**：
```typescript
// 流式响应泛型
interface StreamPayload<T = string> {
    type: 'delta' | 'references' | 'error';
    content?: T;
    references?: Document[];
    message?: string;
}

// 工具调用结果泛型
interface ToolResult<T> {
    success: boolean;
    data?: T;
    error?: string;
}

// 提取 Promise 返回类型
type UnpackPromise<T> = T extends Promise<infer U> ? U : T;
type ChatResponse = UnpackPromise<ReturnType<typeof chatApi.send>>;
```

### 5. CSS 如何实现一个类似 ChatGPT 对话框的自动滚动到底部效果？

**参考答案**：

**ChatGPT 式自动滚动**：新消息到来时自动滚动到底部，但用户向上翻阅时不强制滚动。

**在本项目中的实现**（ChatWindow.jsx）：

```javascript
const AUTO_SCROLL_THRESHOLD = 120;
const shouldAutoScrollRef = useRef(true);

const scrollToBottom = (behavior = 'auto') => {
    if (!shouldAutoScrollRef.current) return;
    const list = messagesListRef.current;
    if (list) {
        list.scrollTo({ top: list.scrollHeight, behavior });
        return;
    }
    messagesEndRef.current?.scrollIntoView({ behavior });
};

const handleMessagesScroll = () => {
    const list = messagesListRef.current;
    if (!list) return;
    // 计算距离底部的距离
    const distanceToBottom = list.scrollHeight - list.scrollTop - list.clientHeight;
    // 只有当距离底部小于阈值时，才自动滚动
    shouldAutoScrollRef.current = distanceToBottom <= AUTO_SCROLL_THRESHOLD;
};

// 监听消息变化，自动滚动
useEffect(() => {
    scrollToBottom(isGenerating ? 'auto' : 'smooth');
}, [messages, isGenerating]);
```

**CSS 配合**：
```css
.messages-list {
    height: calc(100vh - 200px);
    overflow-y: auto;
    /* 平滑滚动 */
    scroll-behavior: smooth;
    /* 滚动条样式 */
    scrollbar-width: thin;
    scrollbar-color: #ccc transparent;
}

.messages-list::-webkit-scrollbar {
    width: 6px;
}

.messages-list::-webkit-scrollbar-thumb {
    background-color: #ccc;
    border-radius: 3px;
}
```

**关键点**：
1. **阈值判断**：用户距离底部 120px 以内时才自动滚动，避免用户向上看时被"拽"回来
2. **两种滚动行为**：生成中用 `auto`（更快），生成完用 `smooth`（更顺滑）
3. **监听滚动事件**：用户手动滚动时更新 `shouldAutoScrollRef`

**备用方案**：
```html
<!-- 用锚点元素 -->
<div class="messages-list">
    <div class="message">...</div>
    <div class="message">...</div>
    <div id="messages-end"></div>
</div>

<script>
document.getElementById('messages-end').scrollIntoView({ behavior: 'smooth' });
</script>
```

### 6. 算法题：给定一个字符串，判断其中的括号（包括大中小括号）是否合法有效。

**参考答案**：

**LeetCode 20. Valid Parentheses**

**题目描述**：
```
给定一个只包括 '('，')'，'{'，'}'，'['，']' 的字符串，判断字符串是否有效。
有效字符串需满足：
1. 左括号必须用相同类型的右括号闭合
2. 左括号必须以正确的顺序闭合

示例 1：
输入：s = "()[]{}"
输出：true

示例 2：
输入：s = "(]"
输出：false
```

**解法：栈（经典题）**

```javascript
function isValid(s) {
    const stack = [];
    const map = {
        ')': '(',
        ']': '[',
        '}': '{'
    };

    for (const char of s) {
        // 如果是右括号
        if (char in map) {
            // 栈为空 或 栈顶不匹配 → 无效
            if (stack.length === 0 || stack.pop() !== map[char]) {
                return false;
            }
        } else {
            // 左括号入栈
            stack.push(char);
        }
    }

    // 栈必须为空才算有效
    return stack.length === 0;
}

// 测试
console.log(isValid('()[]{}'));  // true
console.log(isValid('(]'));      // false
console.log(isValid('([{}])'));  // true
console.log(isValid('('));        // false
```

**时间复杂度**：O(n)，n 是字符串长度
**空间复杂度**：O(n)，最坏情况全是左括号

**边界情况**：
- 空字符串 → true（看题目要求，有时认为 false）
- 只有左括号 → false
- 只有右括号 → false
- 括号顺序不对 → `([)]` → false

**TypeScript 版本**：
```typescript
function isValid(s: string): boolean {
    const stack: string[] = [];
    const closeToOpen: Record<string, string> = {
        ')': '(',
        ']': '[',
        '}': '{'
    };

    for (const char of s) {
        if (char in closeToOpen) {
            if (stack.length === 0 || stack.pop() !== closeToOpen[char]) {
                return false;
            }
        } else {
            stack.push(char);
        }
    }

    return stack.length === 0;
}
```

## 面试总结

**本场考点分布**：
1. **JS 基础**：事件循环、微任务/宏任务
2. **网络通信**：SSE vs WebSocket（AI 应用高频）
3. **异步控制**：Promise 并发控制
4. **TypeScript**：泛型、extends 关键字
5. **CSS/交互**：ChatGPT 式自动滚动
6. **算法**：有效括号（栈）

**前端 AI 应用面试建议**：
- **流式输出是重点**：SSE、ReadableStream、`fetch` + `getReader()` 必须熟练
- **Promise/异步**：并发控制、重试、超时、取消，都是实际项目中会遇到的
- **TypeScript**：泛型、类型体操，AI 项目类型复杂，TS 用得深
- **React**：useRef、useEffect、SSE 解析、自动滚动，都是 Chat 界面必备
